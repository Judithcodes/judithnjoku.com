MY EXPERIENCE WITH LEARNING WEB DEVELOPMENT/ PROGRAMMING

DAY ONE

19/12/2017

I wake up with a start. My heart is thumping, but I have no Idea why. I take a quick look at my clock, its 2:23 am. I jump up immediately. I have work to do and I am 23 minutes late. I don’t want my plan to be a disaster.

This is my first day on Udemy’s full stack web development program. I have downloaded only 50 lessons of the course so far. There are 384 topics in all and I can’t wait to have it all within reach. I spent the first hour of my time, downloading these video lessons. You can never fully trust your internet connection to be stable. It is 3:55am and my eyes feel really tired, but I refuse to stop. I have been battling for months with the idea of combining work with study and I must state that this is not just any kind of work, but something tedious. I am a Customer Experience Management Personnel with Sterling Bank Plc, Nigeria, and the job can get really time and strength consuming. I thought undertaking this program will be an impossible feat and I kept pushing it forward until I will have some free time. I know that time might never come. I will like to quit my job someday and pursue a career in web development. I cannot pretend to see the future, but I believe that I can only achieve what I hope for and work towards. I already have a reputation for sticking with things.

Here I am writing about my daily experience with programming. I have always loved creating things, as long as I have the basic knowledge, I will be inclined to want to create more. My plan is to wake up daily at 2am and study until 4:30 am when I will start preparations for work. Do I get to sleep? Of course. 5 hours of sleep is not so bad. 9pm until 2am. I know that it could mean more stress, but I believe with time, I will get used to it.

What have I learned today?

Today, I studied about the basic structure of a website. Udemy’s tutor for this course does justice to this, by using Wikipedia’s home page. I was able to make meaning of much of the codes I always run into whenever I get curious and begin to look out for source codes of websites. All these HTML tags now make so much sense. HTML stands for Hypertext markup language. Markup languages use tags. Tags are written in between two brackets as depicted below.

Basically when you type a web address, a request is sent to the server where the websites literally lives. If the request is successful, it sends the Html code to be downloaded by your computer. The web browser in use then reads this code and displays it on the screen.

The basic structure of a website is to have your codes in-between two html tags.

The open tag and the close tag which is differentiated by the forward slash in the closing tag. Whenever you open a tag, you must close it. This is a general rule. The HEAD tag defines what you have in the heading section of your web page, while the BODY tag contains all that will be seen in the web page.

The html file starts with an open tag of HTML and ends with a close tag of HTML and tells the web page where to start and end.

COMMENTS

You can write comments inbetween your codes. This will not be visible when you run the code, it is only meant to guide, inform or remind you about some details in your code. It basically makes the code more readable. With these comments another developer can read and understand your work.

META ELEMENTS

This can simply be described as a code that helps to give data about data. It is usually part of the head section. They are used to specify certain description and specifications about the webpage. For example the character set of the webpage can be set to UTF 8.

These are just a few titbits of what I learned today

I am trying not to overwhelm myself, therefore I will take each day slowly, knowing that I only get an hour to learn, when I could actually have 8 hours and achieve more, supposing I didn’t have my job.

I am Judith and I will keep coding for the love of creation.
